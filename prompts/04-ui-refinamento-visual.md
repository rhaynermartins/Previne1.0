# Fase 14 — Refinamento Visual Profundo de UI/UX

Este arquivo deve ser usado como referência obrigatória durante a Fase 14 do projeto `Previne1.0` / `nova-previne`.

O objetivo desta fase é elevar drasticamente a qualidade visual do sistema.

O projeto já possui funcionalidades importantes implementadas. Agora o foco é transformar a interface em algo realmente profissional, moderno, refinado e convincente para uma clínica odontológica real.

Não trate esta fase como pequenos ajustes de espaçamento. Esta fase deve ser uma revisão visual profunda.

---

## Objetivo principal

O site deve deixar de parecer um sistema cru ou acadêmico e passar a parecer um produto real, bem acabado, confiável e pronto para apresentação comercial.

A experiência visual deve transmitir:

* confiança;
* cuidado;
* saúde;
* tecnologia;
* limpeza;
* acolhimento;
* profissionalismo;
* modernidade;
* credibilidade clínica;
* sensação premium.

O resultado deve parecer digno de uma clínica odontológica moderna, não apenas um projeto funcional.

---

## Identidade visual esperada

A identidade da Nova Previne deve ser aplicada de forma consistente.

Use como base:

* azul;
* verde;
* branco;
* tons claros de azul e verde;
* azul escuro para títulos;
* cinzas suaves para textos secundários;
* fundos claros e limpos;
* elementos arredondados;
* ícones modernos;
* sombras suaves;
* gradientes discretos.

A aparência deve lembrar uma clínica odontológica atual, tecnológica e acolhedora.

Evite visual genérico de template.

Evite telas planas demais, sem personalidade.

Evite excesso de elementos decorativos.

Evite parecer panfleto.

A interface deve ser sofisticada, limpa e bem composta.

---

## Referência estética

Use como inspiração as artes criadas para a Nova Previne:

* azul e verde como cores principais;
* destaque para os 20 anos da clínica;
* layout limpo;
* visual odontológico;
* sensação de campanha profissional;
* textos bem hierarquizados;
* elementos curvos suaves;
* aparência clara, confiável e moderna.

Mas adapte isso para site/sistema.

O site não deve parecer um post de Instagram. Ele deve usar a mesma linguagem visual, mas com acabamento de produto web profissional.

---

## Direção visual desejada

A interface deve ter uma estética próxima de:

* clínica odontológica premium;
* healthtech moderna;
* SaaS médico;
* plataforma de agendamento profissional;
* dashboard limpo e elegante;
* landing page institucional de alto nível.

Cada tela deve parecer pensada, não apenas montada.

## Home

A Home deve ser uma das páginas mais fortes visualmente.

Ela deve ter:

* hero section mais impactante;
* título forte e bem posicionado;
* subtítulo claro;
* botões com boa presença visual;
* uso elegante das cores da marca;
* destaque para os 20 anos;
* cards de diferenciais mais modernos;
* seção de tratamentos mais refinada;
* seção de dentistas mais profissional;
* chamada para agendamento mais convincente;
* chamada para WhatsApp mais bem resolvida;
* footer mais confiável e completo.

A Home deve causar boa primeira impressão.

---

## Páginas públicas

As páginas públicas devem parecer institucionais e profissionais.

Melhore:

* espaçamento entre seções;
* hierarquia de títulos;
* largura dos containers;
* cards;
* ícones;
* cabeçalhos internos;
* chamadas para ação;
* legibilidade;
* responsividade.

Cada página deve ter um topo bem definido, com título, subtítulo e contexto.

---

## Dashboards

Os dashboards não podem parecer crus.

Eles devem parecer painéis profissionais de uma plataforma real.

Melhore:

* cards de resumo;
* organização das informações;
* espaçamento;
* sidebar;
* header interno;
* badges;
* listas de consultas;
* cards de pacientes/dentistas;
* ações principais;
* tabelas;
* contraste;
* responsividade.

O dashboard do paciente deve parecer acolhedor e simples.

O dashboard do dentista deve parecer produtivo e clínico.

O dashboard admin deve parecer gerencial, limpo e objetivo.

---

## Fluxo de agendamento

O fluxo de agendamento deve ser visualmente excelente.

Ele é uma das partes principais do sistema.

Melhore:

* etapa atual;
* navegação entre etapas;
* cards de serviço;
* cards de dentista;
* seleção de data;
* seleção de horário;
* campo de descrição do caso;
* tela de confirmação;
* feedback de erro;
* feedback de sucesso;
* experiência mobile.

O paciente precisa entender facilmente onde está, o que deve fazer e qual será o próximo passo.

## Botões

Botões devem ter hierarquia clara.

Melhore:

* botão primário;
* botão secundário;
* botão destrutivo/cancelamento;
* botão de ação positiva;
* estados hover;
* estados disabled;
* estado loading;
* espaçamento;
* bordas;
* transições.

Botões importantes devem parecer clicáveis e confiáveis.

---

## Cards

Cards devem ficar mais premium.

Use:

* bordas suaves;
* sombra leve;
* cantos arredondados;
* espaçamento interno confortável;
* títulos claros;
* ícones discretos;
* separação visual adequada;
* hover sutil quando fizer sentido.

Evite cards simples demais, sem presença visual.

---

## Formulários

Formulários devem ser claros, bonitos e fáceis de preencher.

Melhore:

* labels;
* placeholders;
* mensagens de erro;
* alinhamento;
* espaçamento;
* agrupamento de campos;
* botões;
* estados de foco;
* responsividade;
* feedback de sucesso/erro.

Nenhum formulário deve parecer improvisado.

---

## Estados vazios, loading e erro

Toda tela importante deve ter estados bem tratados.

Melhore:

* tela vazia sem consultas;
* tela vazia sem notificações;
* tela vazia sem mensagens;
* loading de páginas;
* loading de ações;
* mensagens de erro;
* mensagens de sucesso.

Estados vazios devem orientar o usuário.

Exemplo:

Em vez de apenas “Nenhuma consulta encontrada”, use algo como:

“Você ainda não possui consultas agendadas. Quando uma nova consulta for solicitada, ela aparecerá aqui.”

---

## Microcopy

Revise textos do sistema para ficarem mais humanos e profissionais.

Use linguagem clara, amigável e objetiva.

Evite mensagens técnicas demais.

Exemplos de tom:

* “Sua consulta foi solicitada com sucesso.”
* “O dentista ainda precisa confirmar esse horário.”
* “Escolha um horário disponível para continuar.”
* “Descreva brevemente o motivo da consulta.”
* “Nenhuma notificação por enquanto.”
* “Você está em dia por aqui.”

---

## Responsividade

A responsividade deve ser revista com cuidado.

Testar mentalmente e ajustar:

* desktop;
* notebook;
* tablet;
* mobile.

No mobile:

* menus devem funcionar bem;
* cards devem empilhar corretamente;
* botões devem ter largura confortável;
* tabelas devem virar cards ou ter scroll bem tratado;
* textos não devem quebrar de forma estranha;
* formulários devem ser confortáveis;
* dashboards devem continuar utilizáveis.

Não considere a Fase 14 concluída sem revisar mobile.

---

## Acessibilidade

Melhore sempre que possível:

* contraste;
* labels;
* foco visível;
* texto alternativo em imagens;
* botões com nomes claros;
* não depender só de cor para status;
* tamanhos de fonte legíveis;
* área de clique adequada.

---

## O que não fazer nesta fase

Não refazer regra de negócio.

Não refazer autenticação.

Não refazer banco.

Não alterar schema ou migrations sem necessidade real.

Não criar funcionalidades grandes novas.

Não avançar para Fase 15.

Não trocar a arquitetura do projeto.

Não remover funcionalidades existentes.

Não quebrar fluxos já implementados.

Não priorizar beleza sacrificando usabilidade.

---

## Checklist de qualidade visual

Antes de concluir, revise:

* O site parece profissional?
* A interface parece de uma clínica real?
* A identidade azul/verde/branco está consistente?
* A Home causa boa primeira impressão?
* Os dashboards parecem produtos reais?
* O fluxo de agendamento está claro?
* Os botões têm hierarquia?
* Os cards estão bem acabados?
* Os formulários estão confortáveis?
* O mobile está bom?
* Os estados vazios foram tratados?
* As mensagens são humanas?
* As páginas parecem parte do mesmo sistema?
* O visual está acima de um template básico?

Se algum ponto ainda parecer cru, refine antes de finalizar.

## Resultado esperado

Ao final da Fase 14, o sistema deve continuar funcional, mas com aparência muito superior.

A diferença visual antes/depois deve ser clara.

O projeto deve parecer mais moderno, mais premium, mais confiável e mais apresentável.

A interface deve ficar forte o suficiente para uma demonstração real da Clínica Odontológica Nova Previne.
